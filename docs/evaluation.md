# Classic 評価関数 (kishi-to-classic)

Classic エンジン (`kishi-to-classic`) が使用する線形評価関数の詳細。
ソースコードは `src/evaluation.cpp` / `src/evaluation.h`。

## 概要

44 次元の特徴量ベクトルと重みの内積で評価値を算出する線形評価関数。

```
score = clamp(dot(features, weights), -30000, +30000)
```

正のスコアは `perspective`（評価視点の手番）が有利、負は不利を意味する。

### 位置・囲い (42-43)

| # | 内容 | デフォルト重み |
|---|------|--------------|
| 42 | Piece-Square Table (PST) スコア — 段ベースの位置ボーナス合計 | 10 |
| 43 | 玉の歩カバー — 玉の筋±1にある自歩の数の差 | 30 |

**PST**: 駒種ごとに段(相対段: 1=敵陣最深, 9=自陣最後列)に応じたボーナスを加算。歩・桂・銀は前進するほど高評価、飛角は0、成駒は敵陣にいると小ボーナス。

**玉の歩カバー**: 玉のいる筋と隣接筋の自陣の歩の数を Bitboard の `FileMask` と `popcount()` で高速計算。囲いの堅さの近似。

## 重みの学習

### CPU オンライン学習 (`learning.cpp`)

対局終了時 (`gameover win/lose`) に以下の手順で重みを更新する。

1. 対局中に各手を記録: `(局面, 指し手, 手番, エンジンの手か)`
2. 対局終了時、記録した各手について:
   - **chosen**: 実際に指した手の後の局面の特徴量
   - **baseline**: その局面の全合法手の後の特徴量の平均
   - **delta** = chosen - baseline
3. 重み更新:
   ```
   weights += delta * learningRate * actorScale * recencyScale * outcome
   ```
   - `learningRate`: 0.01 (デフォルト)
   - `actorScale`: エンジンの手なら 1.0、相手の手なら 0.65
   - `recencyScale`: 序盤の手 0.35 → 終盤の手 2.0 (終盤の手ほど重視)
   - `outcome`: 勝者の手なら +1、敗者の手なら -1

4. **L2 正則化**: デルタ適用前に `weights *= (1 - 0.001)` で weight decay。重みをゼロ方向へ緩やかに引き戻し、発散を防止。
5. 重みクランプ: 各重みは `±max(|デフォルト重み| * 5, 5)` に制限。  
   L2 正則化との相乗効果で安定した学習を実現。

### 直感的な解釈

- **勝った側の手**: 選んだ手の特徴量が平均より高い方向に重みを動かす (模倣)
- **負けた側の手**: 選んだ手の特徴量が平均より高い方向と逆に重みを動かす (抑制)
- 相手の良い手も学習に取り込む (`actorScale=0.65`)

### GPU 学習 (オプション)

`UseGpu=true` かつ `GpuTrainOnGameEnd=true` のとき、対局終了時に `tools/gpu_eval.py train` を呼び出し、PyTorch の MLP (42→64→32→1) を追加学習する。CPU 学習とは独立に動作する。GPU が利用できない場合は自動的にスキップされる。

## 評価値の範囲

| 範囲 | 意味 |
|------|------|
| -30,000 ~ +30,000 | 通常の評価値 (cp 単位) |
| ±100,000 + depth | 詰み (MateScore) |

探索中、詰みを発見した場合は `±(100000 + depth)` を返す。depth が大きいほど短い詰みを意味し、高く評価される。

## 重みファイル

`random-shogi.weights` はプレーンテキストで、1 行に 1 つの重み値が並ぶ (44 行)。起動時に自動ロードされ、対局終了時に上書き保存される。ファイルが存在しない場合はデフォルト重みが使われる。旧 42 行ファイルも読み込み可能 (不足分はデフォルト重みで補完)。
